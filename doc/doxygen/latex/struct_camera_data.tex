\doxysubsection{Detailed Description}
Data structure for Camera component. 

Following EPOS Smart\+Data principles, this structure contains all the data needed for \doxylink{struct_camera_component}{Camera\+Component} functionality. Generates realistic pixel matrix data for image simulation. Initially simplified to handle EXTERNAL\+\_\+\+PIXEL\+\_\+\+MATRIX only, but can be extended to support RGB\+\_\+\+IMAGE, VIDEO\+\_\+\+STREAM, and CAMERA\+\_\+\+METADATA.

Replaces inheritance-\/based data storage with pure composition.

Sets up the same random distributions as the original \doxylink{struct_camera_component}{Camera\+Component}\+:
\begin{DoxyItemize}
\item Image dimensions\+: 640x480 (VGA resolution, typical for automotive cameras)
\item Bytes per pixel\+: 1 (grayscale) initially, can be extended to 3 (RGB)
\item Pixel values\+: 0-\/255 (standard 8-\/bit grayscale)
\item Noise\+: ±10 (realistic sensor noise)
\item Pattern variation\+: 0.\+0-\/1.\+0 (for generating realistic image patterns)
\item Frame variation\+: 0-\/100 (frame-\/to-\/frame variation)
\item Delay\+: 30-\/40ms (typical camera \texorpdfstring{$\sim$}{\string~}30\+Hz frame rate) 
\end{DoxyItemize}
